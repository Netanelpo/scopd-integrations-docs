    AWS CloudTrail integration with SCOPD SIEM

    1. Create an AWS account

    1. Go to https://aws.amazon.com/
    2. Click Create an AWS Account.

    3. Fill in:

    4. Email

    5. Password
    6. Phone number
    7. Credit/debit card (required for Free Tier)
    8. After registration, log in to the AWS Console: console.aws.amazon.com

    9. In the top-right corner choose a region — for example:

    10. eu-north-1 (Stockholm)

    2. Create an S3 bucket for CloudTrail logs

    1. In the AWS search bar at the top type S3 and open the service.
    2. Click Create bucket.

    3. Fill in:

    4. Bucket name: scopd-cloudtrail-logs-<name>

    5. Region: the same region you selected above
    6. Keep public access blocked.
    7. Leave other settings as default.
    8. Done.

    3. Configure CloudTrail to send logs to S3

    1. In the AWS search bar type CloudTrail → open Trails.
    2. Click Create trail.

    3. Fill in:

    4. Trail name: scopd-cloudtrail-trail-<name>

    5. In Storage location choose:

    6. Use existing bucket → select your bucket (from the previous section).

    7. Multi-region trail — you can keep it enabled.
    8. Log file SSE-KMS encryption — uncheck this option.
    9. Click Create trail.

    Now CloudTrail will store .json.gz files in your S3 bucket.

    4. Create an IAM policy for SIEM

    SCOPD SIEM must have permissions to:

    • s3:ListBucket

    • s3:GetObject

    • Open IAM → Policies → Create policy.

    • Select the JSON tab.
    • Paste the following:
    Show IAM policy JSON 
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::scopd-cloudtrail-logs-<name>"
    },
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::scopd-cloudtrail-logs-<name>/*"
    }
  ]
}
    1. Name the policy: ScopdCloudTrailAccess
    2. Create the policy.

    5. Create an IAM user and access keys

    1. Go to IAM → Users → Create user.
    2. User name: scopd-aws-reader

    3. In Permissions choose Attach policies directly and attach:

    4. ScopdCloudTrailAccess

    5. Finish user creation.
    6. On the user page, click Create access key.

    Save the Access key ID and Secret access key — you will need them later.

    6. Configure AWS credentials on the SIEM Manager

    On the SIEM server:

    sudo su -
mkdir -p /root/.aws
nano /root/.aws/credentials

    Paste:

    [scopd-aws]
aws_access_key_id = <your AccessKeyID>
aws_secret_access_key = <your SecretAccessKey>

    Save the file.

    Then create the config file:

    nano /root/.aws/config

    [profile scopd-aws]
region = eu-north-1
output = json

    7. Install AWS CLI (optional, for testing)

    Installation commands for AWS CLI 
    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
    Verification: 
    /usr/local/bin/aws --version

    8. Test the AWS profile

    /usr/local/bin/aws sts get-caller-identity --profile scopd-aws

    If you see UserId and Arn in the output, the profile works correctly.

    9. Test access to the S3 bucket

    /usr/local/bin/aws s3 ls s3://scopd-cloudtrail-logs-<name> --profile scopd-aws

    If you see something like:

    PRE AWSLogs/

    → access to the bucket is working.

    10. Configure SIEM for AWS S3

    Open the SCOPD SIEM configuration:

    nano /var/ossec/etc/ossec.conf

    Add the following before </ossec_config>:

    aws-s3 wodle configuration 
    <wodle name="aws-s3">
  <disabled>no</disabled>
  <interval>30m</interval>
  <run_on_start>yes</run_on_start>
  <skip_on_error>no</skip_on_error>
  <bucket type="cloudtrail">
    <name>wazuh-cloudtrail-logs-<name></name>
    <aws_profile>scopd-aws</aws_profile>
  </bucket>
</wodle>

    Save the file.

    11. Restart SIEM Manager

    sudo systemctl restart wazuh-manager

    12. Check Wazuh logs

    sudo tail -f /var/ossec/logs/ossec.log | grep aws

    Expected log lines:

    INFO: Starting fetching of logs
INFO: Executing Bucket Analysis
INFO: Fetching logs finished

    There should be no error messages.

    13. Check that CloudTrail events appear

    CloudTrail writes files to S3 with a delay of about 1–5 minutes. Check:

    /usr/local/bin/aws s3 ls s3://scopd-cloudtrail-logs-<name>/AWSLogs/ --recursive --profile scopd-aws

    If you see .json.gz files → SCOPD SIEM will process them.

    14. Check in the Wazuh Dashboard

    Open the Amazon Web Services section in the SIEM dashboard.

    Integration is complete — CloudTrail events are now ingested into SCOPD SIEM.
