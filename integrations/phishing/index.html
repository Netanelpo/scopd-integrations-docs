    SCOPD Phishing attack simulator Integration

    Setting up SIEM Rules

    Put the following rule definition inside the <rules> section in /var/ossec/etc/rules/scopd_rules.xml

    <group name="phishing,scopd,">
  <rule id="100180" level="12">
    <decoded_as>json</decoded_as>
    <if_sid>86600</if_sid>
    <field name="event_type">^phishing_link_click$</field>
    <description>Phishing Simulation: User clicked on a phishing link. Email: $(user_email)</description>
    <options>no_full_log</options>
  </rule>
</group>

    Restart SIEM Manager

    sudo systemctl restart wazuh-manager

    The Phishing Simulator agent

    !Attention! On this machine, you need both the SIEM agent installed and the Phishing Simulator client itself!

    • Agent file: /var/ossec/etc/ossec.conf

    Add one block: 

    <ossec_config>

  <localfile>

    <log_format>syslog</log_format>

    <location>/var/log/syslog</location>

  </localfile>

</ossec_config>

    Restart Linux Agent

    sudo systemctl restart wazuh-agent
